# 动漫记录网站 — 数据库与后端设计方案

## Context

从零开始搭建一个**多用户**动漫记录平台，每个用户独立管理自己的观看记录和收藏。

核心功能：
- **多用户**系统，每个用户独立管理自己的观看记录和收藏
- **中央动漫库**，用户从库中选择动漫，也可自行提交（需管理员审核）
- 动漫支持**中文名、日文名**，以及**标签**分类
- 封面图**存储于本地**，命名为 `{animeId}_{UUID}.{ext}`
- **按天记录观看**：哪天看了哪部动漫的哪几集
- **默认收藏夹**替代状态字段：想看/在看/看过/搁置/抛弃 作为系统自动创建的收藏夹
- 用户可**自定义收藏夹**进行分类整理

## 数据库设计（共 7 张表）

### 1. `user` — 用户表

```sql
CREATE TABLE user (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) NOT NULL COMMENT '显示名称/昵称',
  account VARCHAR(50) NOT NULL COMMENT '登录账号',
  email VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
  password VARCHAR(255) NOT NULL COMMENT 'BCrypt hash',
  phone_number VARCHAR(20) DEFAULT NULL COMMENT '手机号',
  role VARCHAR(20) DEFAULT 'user' COMMENT 'user/admin',
  avatar_url VARCHAR(500) DEFAULT NULL COMMENT '头像路径',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_account (account)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INT PK | 自增主键 |
| username | VARCHAR(50) | 显示名称/昵称 |
| account | VARCHAR(50) UNIQUE | 登录账号 |
| email | VARCHAR(100) | 邮箱 |
| password | VARCHAR(255) | BCrypt 加密 |
| phone_number | VARCHAR(20) | 手机号 |
| role | VARCHAR(20) | user / admin |
| avatar_url | VARCHAR(500) | 头像路径 |
| created_at / updated_at | DATETIME | 时间戳 |

### 2. `anime` — 动漫库

```sql
CREATE TABLE anime (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name_cn VARCHAR(200) NOT NULL COMMENT '中文名',
  name_jp VARCHAR(200) DEFAULT NULL COMMENT '日文名',
  broadcast_time VARCHAR(50) COMMENT '放送日期，如 2026-07-09',
  bangumi_score VARCHAR(10) COMMENT 'Bangumi评分',
  episode INT DEFAULT 0 COMMENT '总集数',
  state VARCHAR(20) COMMENT '放送状态：已完结/连载中/未开播',
  cover_url VARCHAR(500) COMMENT '封面图本地路径 covers/{animeId}_{UUID}.jpg',
  description TEXT COMMENT '简介',
  -- 审核相关
  review_status VARCHAR(20) DEFAULT 'approved' COMMENT 'pending/approved/rejected',
  submitted_by INT COMMENT '提交者用户ID',
  reviewed_by INT COMMENT '审核者管理员ID',
  review_comment VARCHAR(500) COMMENT '审核意见',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_name_cn (name_cn),
  INDEX idx_name_jp (name_jp),
  INDEX idx_review_status (review_status),
  FOREIGN KEY (submitted_by) REFERENCES user(id),
  FOREIGN KEY (reviewed_by) REFERENCES user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='动漫库表';
```

> **封面图说明**：上传后存于 `{uploadDir}/covers/{animeId}_{UUID}.{ext}`，数据库中只存相对路径。访问方式：`/api/file/covers/{animeId}_{UUID}.{ext}`。

### 3. `tag` / `anime_tag` — 动漫标签

```sql
CREATE TABLE tag (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50) NOT NULL UNIQUE COMMENT '标签名，如 热血/恋爱/科幻'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='标签表';

CREATE TABLE anime_tag (
  id INT AUTO_INCREMENT PRIMARY KEY,
  anime_id INT NOT NULL,
  tag_id INT NOT NULL,
  UNIQUE KEY uk_anime_tag (anime_id, tag_id),
  FOREIGN KEY (anime_id) REFERENCES anime(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tag(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='动漫标签关联表';
```

> 标签为预定义集合，管理员可管理。动漫与标签为多对多关系。

**预置标签：**
```sql
INSERT INTO tag (name) VALUES
  ('热血'), ('恋爱'), ('科幻'), ('奇幻'), ('搞笑'),
  ('日常'), ('治愈'), ('悬疑'), ('战斗'), ('校园'),
  ('异世界'), ('后宫'), ('百合'), ('美食'), ('运动');
```

### 4. `watch_log` — 观看日志

```sql
CREATE TABLE watch_log (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  anime_id INT NOT NULL,
  watch_date DATE NOT NULL COMMENT '观看日期',
  ep_start INT DEFAULT NULL COMMENT '起始集数',
  ep_end INT DEFAULT NULL COMMENT '结束集数',
  ep_count INT DEFAULT 1 COMMENT '看了几集',
  notes TEXT COMMENT '当次笔记/感想',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_date (user_id, watch_date),
  INDEX idx_anime (anime_id),
  FOREIGN KEY (user_id) REFERENCES user(id),
  FOREIGN KEY (anime_id) REFERENCES anime(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='观看日志表';
```

> **核心设计**：每条记录 = "某用户在某天看了某部动漫的第 X~Y 集，共 N 集"。一天看多次同一部动漫可以多条记录，也支持合并为一条（ep_start=1, ep_end=3, ep_count=3）。

### 5. `collection` — 收藏夹

```sql
CREATE TABLE collection (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  name VARCHAR(100) NOT NULL COMMENT '收藏夹名称',
  description VARCHAR(500) DEFAULT NULL COMMENT '收藏夹描述',
  is_default TINYINT(1) DEFAULT 0 COMMENT '是否为系统默认收藏夹',
  sort_order INT DEFAULT 0 COMMENT '排序',
  is_public TINYINT(1) DEFAULT 0 COMMENT '是否公开',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user (user_id),
  FOREIGN KEY (user_id) REFERENCES user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收藏夹表';
```

> **默认收藏夹**：每个用户注册时自动创建 5 个默认收藏夹（is_default=1）：
> - 想看
> - 在看
> - 看过
> - 搁置
> - 抛弃
>
> 用户移动动漫到这些收藏夹即表示更新观看状态。用户在动漫详情页点"想看"就自动加入"想看"默认收藏夹。

### 6. `collection_item` — 收藏条目

```sql
CREATE TABLE collection_item (
  id INT AUTO_INCREMENT PRIMARY KEY,
  collection_id INT NOT NULL,
  anime_id INT NOT NULL,
  added_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_collection_anime (collection_id, anime_id),
  FOREIGN KEY (collection_id) REFERENCES collection(id) ON DELETE CASCADE,
  FOREIGN KEY (anime_id) REFERENCES anime(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收藏条目表';
```

> 一部动漫可以同时在一个用户的多个收藏夹中（但不能在同一收藏夹中重复）。

### 7. 表关系总览

```
user ──1:N──> watch_log ──N:1──> anime
user ──1:N──> collection ──1:N──> collection_item ──N:1──> anime
anime ──1:N──> anime_tag ──N:1──> tag
```

## 封面图存储方案

- 上传目录：`{project}/uploads/covers/`
- 命名规则：`{animeId}_{UUID}.{ext}`，如 `42_a1b2c3d4.jpg`
- 数据库中 `anime.cover_url` 存：`covers/42_a1b2c3d4.jpg`
- 访问方式：通过 Controller 暴露 `/api/file/{path}` 读取本地文件
- 在 `application.yml` 中配置：
  ```yaml
  app:
    upload:
      path: ./uploads
      cover-dir: covers
  ```

> **使用本地文件存储**。如果后续需要 OSS，只需改存储层。

### Phase 6：安全增强

1. 引入 `spring-security-crypto`，使用 `BCryptPasswordEncoder`
2. 注册：`password = BCrypt.encode(rawPassword)` 存库
3. 登录：`BCrypt.matches(rawPassword, dbPassword)` 验证
4. 管理员端点检查 `user.role == 'admin'`（拦截器或 AOP）

## 核心业务流程

### 用户添加动漫到"想看"
```
POST /api/collection/{defaultWannaWatchId}/add  { animeId: 42 }
→ 自动加入"想看"默认收藏夹（无需创建 watch_log）
```

### 用户标记"正在看"
```
PUT /api/collection/move/42  { fromCollectionId: 想看ID, toCollectionId: 在看ID }
→ 从"想看"移到"在看"
```

### 用户记录观看（核心操作）
```
POST /api/watch/log
{
  animeId: 42,
  watchDate: "2026-06-20",
  epStart: 7,
  epEnd: 9,
  epCount: 3,
  notes: "这集打斗作画很棒"
}
→ 创建一条观看日志
→ 同时自动将动漫加入/保持在"看过"默认收藏夹（若尚未）
```

### 用户自定义收藏
```
POST /api/collection/create  { name: "京阿尼精选", description: "我最爱的京阿尼作品" }
POST /api/collection/{customId}/add  { animeId: 42 }
→ 动漫可以同时在"看过"默认收藏夹和"京阿尼精选"自定义收藏夹中
```
